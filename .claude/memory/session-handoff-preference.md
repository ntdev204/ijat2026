When the user asks to save a session, create a dated session handoff file and keep the saved memory focused on durable, non-transient context.

**Why:** The user explicitly asked to save the session and memory after a multi-step fix/review session, so future handoffs should preserve only the useful long-term context instead of rehashing temporary task details.

**How to apply:** At the end of complex work sessions, write a dated session file, avoid storing transient debugging churn, and keep any memory entry concise and reusable for the next session.
